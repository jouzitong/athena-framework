= 安全测试模块启动与执行方案

== 1. 文档目标

本文档用于指导 `athena-framework-starters-test-security` 的本地启动、功能执行与结果验收，覆盖认证、鉴权、菜单权限控制与用户管理测试。

== 2. 模块信息

* 模块路径：`athena-framework-test/athena-framework-starters-test-security`
* 启动类：`org.athena.test.security.SecurityTestApplication`
* 默认端口：`18090`
* 模板测试台页面：`http://127.0.0.1:18090/template/test`

== 3. 环境要求

* JDK 17+
* Maven 3.8+
* MySQL 8.x（本地可连接）

== 3.1 数据库配置

测试模块默认连接：

* 数据库：`test_security_db`
* 地址：`jdbc:mysql://localhost:3306/test_security_db?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=UTC`
* 用户：`root`
* 密码：`root`

启动前请先创建数据库：

[source,sql]
----
CREATE DATABASE IF NOT EXISTS test_security_db CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
----

应用启动时会执行 `schema.sql` 和 `data.sql` 初始化测试表与样例数据。

=== 3.2 导入 JPA 测试数据库模块

如果需要复用 `athena-framework-test-jpa` 的数据库环境（`jpa_test`），直接在 `application.yml` 中切换数据源地址：

* 数据库：`jpa_test`
* 地址：`jdbc:mysql://localhost:3306/jpa_test?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=UTC`
* 配置文件：`src/main/resources/application.yml`

启动前请先创建数据库：

[source,sql]
----
CREATE DATABASE IF NOT EXISTS jpa_test CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
----

== 4. 测试账号

* `admin/admin123`：`user:read,user:create,menu:view`
* `operator/op123`：`user:read,menu:view`
* `guest/guest123`：无管理权限

== 5. 启动方式

=== 5.1 编译（建议先执行）

[source,bash]
----
mvn -pl athena-framework-test/athena-framework-starters-test-security -am -DskipTests compile
----

=== 5.2 默认启动（full 场景）

[source,bash]
----
mvn -pl athena-framework-test/athena-framework-starters-test-security -am spring-boot:run
----

=== 5.3 单文件场景切换（命令行覆盖）

[source,bash]
----
# S1: 仅认证（local token，关闭 authorization）
mvn -pl athena-framework-test/athena-framework-starters-test-security -am spring-boot:run \
  -Dspring-boot.run.arguments="--athena.security.token.type=local,--athena.security.token.jwt.enabled=false,--athena.security.authorization.enabled=false"

# S3/S4: 鉴权与菜单场景（默认配置即可）
mvn -pl athena-framework-test/athena-framework-starters-test-security -am spring-boot:run

# S5/S7.1: token 配置与模块不匹配（预期启动失败）
mvn -pl athena-framework-test/athena-framework-starters-test-security -am spring-boot:run \
  -Dspring-boot.run.arguments="--athena.security.token.type=redis,--athena.security.token.redis.enabled=true"

# 复用 JPA 测试数据库（jpa_test）
mvn -pl athena-framework-test/athena-framework-starters-test-security -am spring-boot:run \
  -Dspring-boot.run.arguments="--spring.datasource.url=jdbc:mysql://localhost:3306/jpa_test?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=UTC"
----

== 6. 执行方式

=== 6.1 页面测试台执行（推荐）

打开：`http://127.0.0.1:18090/template/test`

页面能力：

* 一键登录 admin/operator/guest
* Token 展示、清理、登出
* 一键执行核心场景：
  - `S1` 未登录访问 `GET /profile/me`（预期 401）
  - `S2` 登录后访问 `GET /profile/me`（预期 200）
  - `S3` 访问 `GET /admin/users`（admin 200，guest 403）
  - 用户管理：`POST /admin/users`（admin 200，operator/guest 403）
  - `S4` 菜单：`GET /menu/current`（按权限返回菜单）
* 控制台输出每次请求的状态码和响应体

=== 6.2 HTTP 文件执行

* `http/auth.http`
* `http/authorization.http`
* `http/menu.http`
* `http/audit.http`

== 7. 测试场景与验收标准

=== 7.1 S1 最小认证

* 操作：未携带 token 访问 `GET /profile/me`
* 预期：401

=== 7.2 S2 JWT 认证

* 操作：登录后访问 `GET /profile/me`
* 预期：200，返回 `subject/authn/session` 等上下文信息

=== 7.3 S3 鉴权

* 操作：
  - `guest` 访问 `GET /admin/users`
  - `admin` 访问 `GET /admin/users`
* 预期：`guest=403`，`admin=200`

=== 7.4 用户管理权限

* 操作：调用 `POST /admin/users`
* 预期：
  - `admin=200`
  - `operator/guest=403`

=== 7.5 S4 菜单权限控制

* 操作：调用 `GET /menu/current`
* 预期：
  - `admin` 返回 `menu:view + user:read + user:create` 相关菜单
  - `operator` 返回 `menu:view + user:read`
  - `guest` 返回空或无管理菜单

=== 7.6 S5 缺失模块失败场景

* 操作：使用 `missing-module` profile 启动（配置 `token.type=redis` 但未引入 redis token 模块）
* 预期：启动失败，日志出现 token 模块缺失/不匹配提示

=== 7.7 审计日志

* 操作：执行登录、访问鉴权接口后，调用 `GET /audit/logs`
* 预期：
  - 认证成功/失败写入 `AUTHN`
  - 权限校验写入 `AUTHZ`

== 8. 关键配置文件

* `src/main/resources/application.yml`（默认）
  - 已合并认证/鉴权/菜单/缺失模块/数据库切换场景说明，可直接作为框架接入参考模板

== 9. 常见问题排查

=== 9.1 端口占用

* 现象：启动失败提示 18090 占用
* 处理：释放端口或修改 `application*.yml` 中 `server.port`

=== 9.2 登录成功但接口 401

* 检查请求头：`Authorization: Bearer <token>`
* 检查是否点了页面上的 `Clear Token`

=== 9.3 期望 403 但返回 200

* 确认当前登录用户角色是否正确
* 确认启动 profile 是否启用 `athena.security.authorization.enabled=true`

=== 9.4 命令行覆盖参数未生效

* 检查启动命令是否带：`-Dspring-boot.run.arguments="--key=value,..."`
* 检查参数分隔是否为英文逗号 `,`

== 10. 建议执行顺序

1. `S1`（未登录 401）
2. `S2`（登录后上下文）
3. `S3`（admin/guest 鉴权差异）
4. 用户管理 `POST /admin/users`（create 权限）
5. `S4`（菜单权限）
6. `S5`（缺失模块失败场景）
