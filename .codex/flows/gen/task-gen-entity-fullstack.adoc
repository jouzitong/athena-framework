== 流程说明

本流程用于生成“新实体 + 全栈交付”的任务定义，
覆盖需求文档、概要设计、详细设计、后端与前端实现等完整链路。
输出为符合《Codex Task 定义规范》的 Task 文件。

== 流程基本信息（Flow Identity）

.flow.identity
[cols="1,3"]
|===
|属性 |值


|flow.id（流程ID）
|TASK_GEN_ENTITY_FULLSTACK

|flow.name（流程名称）
|新实体全栈任务生成流程

|flow.version（流程版本）
|1.0

|flow.owner（流程负责人）
|平台研发团队

|flow.description（流程说明）
|为新实体建设生成完整可执行任务清单
|===

== 进入条件（Entry Conditions）

.entry.conditions
* 已提供需求背景或业务目标
* 已明确目标实体与核心能力边界
* 已确认主要影响范围或待评估范围

.entry.on_fail
[source,text]
----
STOP_AND_REPORT
----

== 任务生成步骤（Generation Steps）

=== Step 1：需求理解与实体边界澄清（Analyze）

.step.1
[cols="1,3"]
|===
|属性 |说明

|step.id
|ANALYZE

|step.type
|analyze

|step.goal
|明确实体定义、业务能力与交付目标

|step.output
|实体清单 + 能力边界 + 关键约束
|===

要求：
- 实体边界必须可验证、可裁决
- 记录所有关键约束与依赖

---

=== Step 2：范围与拆分策略（Scope & Split）

.step.2
[cols="1,3"]
|===
|属性 |说明

|step.id
|SCOPE_SPLIT

|step.type
|plan

|step.goal
|确定模块范围并设计子任务拆分

|step.output
|Task Scope 草案 + 子任务拆分清单
|===

要求：
- Scope 必须包含“包含项/排除项”
- 子任务拆分必须可独立执行

---

|===

|step.output
|Task Overview 内容
|===

---

=== Step 4：Task Meta 生成（Meta）

.step.4
[cols="1,3"]
|===
|属性 |说明

|step.id
|META

|step.type
|generate

|step.goal
|生成任务身份与目标字段

|step.output
|Task Meta + Task Objective
|===

---

=== Step 5：Sub Task 生成（Sub Tasks）

.step.5
[cols="1,3"]
|===
|属性 |说明

|step.id
|SUB_TASKS

|step.type
|generate

|step.goal
|生成完整子任务块与依赖

|step.output
|Sub Task Blocks（含 Objective 与 Completion Policy）
|===

推荐子任务集合：
- 需求文档编写（DOCS）
- 概要设计（DOCS）
- 详细设计（DOCS）
- 后端实现（FUNC）
- 前端实现（FUNC）
- 测试与验收（TEST）
- 运维与文档更新（DOCS，视需要）

依赖建议：
- 概要设计依赖需求文档
- 详细设计依赖概要设计
- 后端与前端实现依赖详细设计
- 测试与验收依赖前后端实现

---

== 退出条件（Exit Criteria）

.exit.conditions
* Task 文件结构完整
* 所有必填字段齐全
* Objective 与 Completion Policy 可验证

== 流程强约束（Mandatory Rules）

.flow.rules
* 不生成实现步骤或代码
* 必须保持 Task 定义结构完整
* 子任务依赖必须显式列出
