== 流程说明

本流程用于生成“重构/迁移类”任务定义，
强调范围约束、回滚策略与验证路径。
输出为符合《Codex Task 定义规范》的 Task 文件。

== 流程基本信息（Flow Identity）

.flow.identity
[cols="1,3"]
|===
|属性 |值


|flow.id（流程ID）
|TASK_GEN_REFACTOR_MIGRATION

|flow.name（流程名称）
|重构与迁移任务生成流程

|flow.version（流程版本）
|1.0

|flow.owner（流程负责人）
|平台研发团队

|flow.description（流程说明）
|为重构/迁移场景生成可裁决的任务定义
|===

== 进入条件（Entry Conditions）

.entry.conditions
* 已提供重构/迁移目标与收益说明
* 已识别影响范围与风险点
* 已明确回滚或止损策略（可在 Risks & Notes 中体现）

.entry.on_fail
[source,text]
----
STOP_AND_REPORT
----

=== Step 3：拆分与依赖（Split）

.step.3
[cols="1,3"]
|===
|属性 |说明

|step.id
|SPLIT

|step.type
|plan

|step.goal
|拆分为可验证的子任务

|step.output
|子任务拆分与依赖
|===

推荐子任务集合：
- 现状评估与基线记录（DOCS）
- 迁移/重构方案设计（DOCS）
- 核心实现与替换（REFACTOR 或 FUNC）
- 回滚与降级策略（DOCS）
- 测试与回归验证（TEST）
- 文档与发布说明（DOCS）

== 退出条件（Exit Criteria）

.exit.conditions
* 任务目标与边界明确
* 回滚/降级策略已体现
* 任务完成条件可验证

== 流程强约束（Mandatory Rules）

.flow.rules
* 不输出实现步骤或代码
* 回滚或止损策略必须显式记录
* Scope 的排除项必须明确
