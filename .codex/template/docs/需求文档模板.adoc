===== 基本信息

===== 功能概述

用 **1~3 段话** 清晰说明：
* 这个功能是做什么的
* 它在系统中的定位
* 它不等同于哪些功能（避免误解）


===== 功能范围（In Scope）

明确 **本功能包含的能力**：

* 功能点 1
* 功能点 2
* 功能点 3


====== 非功能范围（Out of Scope）

明确 **本功能不做的事情**，防止需求膨胀：

* 不支持……
* 不包含……
* 不考虑……

===== 功能需求详述

====== 功能点一：{功能点名称}

.功能说明
描述该功能点的作用和行为。

.操作说明
* 支持的操作
* 操作触发条件
* 操作结果

.输入与输出
* 输入参数说明
* 输出结果说明


====== 功能点二：{功能点名称}

（结构同上，可按需扩展）


===== 异常与边界情况

* 非法输入
* 状态冲突
* 并发场景
* 系统异常时的处理策略


===== 验收标准

* 功能点是否完整实现
* 边界与异常是否符合预期
* 是否满足非功能需求
* 是否通过相关测试

===== 相关说明

* 依赖功能
* 相关文档
* 后续扩展方向
