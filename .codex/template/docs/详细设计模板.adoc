== 核心流程设计

=== 主流程（时序）

1. 接收请求
2. 参数校验
3. 业务处理
4. 数据持久化
5. 返回结果

.具体时序图（含异常分支）
[plantuml, core-sequence-alt, png]
----
@startuml
actor Client
participant Controller
participant Service
participant Repository

Client -> Controller : 发起请求
Controller -> Controller : 参数校验

alt 参数校验失败
  Controller --> Client : 返回参数错误
else 校验通过
  Controller -> Service : 调用业务处理
  alt 业务校验失败
    Service --> Controller : 业务异常
    Controller --> Client : 返回业务错误
  else 业务成功
    Service -> Repository : 数据持久化
    Repository --> Service : OK
    Service --> Controller : 业务结果
    Controller --> Client : 返回成功响应
  end
end
@enduml
----

=== 异常流程

* 参数校验失败
* 业务规则不满足
* 外部依赖异常
