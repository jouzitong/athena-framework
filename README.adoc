== 概述
Athena Framework 是一个面向 Java 应用体系的 *基础能力框架*，
用于沉淀通用工程能力、统一技术规范，并为业务系统提供稳定、可演进的技术底座。

== 项目定位

Athena Framework *不是业务系统*，也不是应用脚手架。

它的定位是：

[quote]
____
为业务系统提供基础能力与工程规范的框架层
____

在整体系统架构中，Athena Framework 位于：

* 业务系统之下
* 通用技术框架之上

它关注的是 *工程一致性*、*长期演进能力* 以及 *团队协作效率*，
而不是具体业务实现。

== 设计目标

Athena Framework 旨在解决以下共性问题：

* 多项目之间如何复用通用工程能力
* 如何统一团队的技术规范与工程约定
* 如何降低新项目的启动与维护成本
* 如何保证系统在长期演进中的稳定性

== 核心理念

=== 分层清晰，职责单一

* 框架只负责 *通用能力*
* 业务系统只负责 *业务逻辑*
* 框架中不引入任何业务语义


=== 低侵入、可选用

* 框架能力以 *可选* 的方式提供
* 不强制业务系统遵循复杂的继承或配置
* 支持渐进式引入与演进

=== 约定优于配置

* 提供清晰的工程约定
* 降低配置与理解成本
* 保留足够的扩展空间

== 使用方式概览

Athena Framework 以 *依赖引入* 的方式被业务系统使用。

对于业务开发者而言：

* 不需要运行 Athena Framework
* 不需要关心其内部实现
* 只需要按约定使用其提供的能力

详细使用方式将在后续文档中说明。

== 适用场景

Athena Framework 适用于：

* 多模块或多服务的业务系统
* 需要长期维护与持续演进的应用
* 希望统一工程规范的团队
* 需要沉淀技术资产的组织

== 项目结构（抽象视角）

[source,text]
----
athena-framework
├── framework-dependencies
├── framework-test
└── pom.xml

说明：

具体模块职责与实现细节不在本说明中展开。

== 快速开始

.如果你是第一次接触 Athena Framework，建议按以下顺序阅读文档：
. 阅读《总体说明》
. 查看《快速开始》
. 根据项目需要引入并使用框架能力

.在你的业务系统的 `pom.xml` 中引入以下依赖：
[source, xml]
----
<dependencyManagement>
    <dependencies>
        <dependency>
            <artifactId>framework-dependencies</artifactId>
            <groupId>org.athena</groupId>
            <version>1.1.0</version> <!-- 选择最新的版本 -->
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
----

后面根据自己的需要引入具体模块。

.例如:
[source, xml]
----
<dependency>
    <groupId>org.athena</groupId>
    <artifactId>athena-framework-common</artifactId>
</dependency>
----


== 总结

Athena Framework 的目标不是做更多功能，
而是：

* 让业务系统更专注
* 让工程结构更一致
* 让技术演进更可控
