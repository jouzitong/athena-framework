= Athena Framework 安全认证授权模块设计
:toc: left
:toclevels: 3
:sectnums:

== 1. 文档说明

=== 1.1 目标

.构建一个可插拔的 Spring Boot 安全框架，支持：
* 用户认证（登录、token）
* 权限控制（接口 / 方法）
* 菜单控制（前端展示）
* 角色（可选）
* 数据权限（预留）
* 审计日志

=== 1.2 设计原则

* 可插拔（starter 方式接入）
* 模块解耦（按能力拆分）
* 角色非必选
* 权限统一以 permissionCode 为核心
* 支持覆盖默认实现
* 支持渐进式扩展

=== 2.2 模块职责说明

[cols="1,3"]
|===
|模块 |职责

|athena-framework-common
|基础工具、常量、异常、通用模型

|athena-framework-starter-security-data-scope
|数据权限（预留实现与扩展 SPI）

|framework-dependencies
|BOM 与版本依赖统一管理

|athena-framework-test
|各 starter 集成验证与示例工程

=== 2.4 模块启停原则

功能模块生效需同时满足两个条件：

* 条件1：模块依赖在 classpath 中（已引入对应 starter）
* 条件2：配置项 `enabled=true`

即“依赖存在 + 配置开启”双重门控，避免误启用。

== 3. 核心设计

=== 3.1 统一认证模型

[source,java]
----
public class LoginUser {
private Long userId;
private String username;
private String tenantId;
private Set<String> permissions;
}
----

=== 3.2 核心接口定义

==== 用户加载

[source,java]
----
public interface AuthUserService {
LoginUser loadByUsername(String username);
LoginUser loadByUserId(Long userId);
}
----

==== 权限服务

[source,java]
----
public interface PermissionService {
    Set<String> getPermissions(Long userId);
}
----

==== 菜单服务

[source,java]
----
public interface MenuService {
    List<MenuNode> getMenus(Long userId);
}
----

==== Token 服务

[source,java]
----
public interface TokenService {
    String createToken(LoginUser user);
    LoginUser parseToken(String token);
    void invalidate(String token);
}
----

==== 数据权限（预留）

[source,java]
----
public interface DataPermissionService {
    DataScope getDataScope(Long userId, String permission);
}
----

== 4. 认证设计

=== 4.1 登录流程

[source,text]
----
用户请求登录
↓
AuthUserService 查询用户
↓
校验密码
↓
生成 LoginUser
↓
TokenService 生成 token
↓
返回 token
----

=== 4.2 Token 方案

支持两种模式：

[cols="1,2"]
|===
|类型 |说明

|JWT
|无状态，简单

|Redis Token
|支持踢下线、实时失效
|===

配置：

[source,yaml]
----
athena:
  security:
    token:
      type: jwt
----

== 5. 授权设计

=== 5.1 权限模型

统一使用：

[source,text]
----
permissionCode

示例：

* user:create
* user:update
* order:list
----

=== 5.2 鉴权方式

==== URL 鉴权

[source,java]
----
/api/user/** -> user:read
----

==== 方法鉴权

[source,java]
----
@RequirePermission("user:create")
----

或：

[source,java]
----
@PreAuthorize("hasAuthority('user:create')")
----

== 8. 自动装配设计

=== 8.1 自动配置类

[source,java]
----
@AutoConfiguration
@ConditionalOnProperty(prefix = "athena.security.auth", name = "enabled", havingValue = "true", matchIfMissing = true)
public class SecurityAuthCoreAutoConfiguration {
}
----

可选模块示例（以菜单模块为例）：

[source,java]
----
@AutoConfiguration
@ConditionalOnClass(MenuService.class)
@ConditionalOnBean(SecurityAuthCoreMarker.class)
@ConditionalOnProperty(prefix = "athena.security.menu", name = "enabled", havingValue = "true", matchIfMissing = true)
public class SecurityMenuAutoConfiguration {
}
----

=== 8.2 默认 Bean 策略

[source,text]
----
提供默认实现
↓
业务可覆盖
----

示例：

[source,java]
----
@ConditionalOnMissingBean
@Bean
public TokenService tokenService() {
    return new JwtTokenService();
}
----

=== 8.3 自动装配注册

每个子模块独立维护自己的：

`META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`

这样可以做到子模块解耦发布与独立开关。

== 9. Starter 设计

=== 9.1 基础 starter

沿用现有 starter 组合方式，安全能力改为“核心必选 + 功能按需”。

推荐基础依赖组合：

* `athena-framework-starter-web`
* `athena-framework-starter-data`（按需再选 jdbc/jpa/mybatis/mongo）
* `athena-framework-starter-websocket`（按需）

=== 9.2 安全 starter

拆分策略：

* 必选：`athena-framework-starter-security-auth-core`
* 可选：`token-jwt`、`token-redis`、`authorization`、`menu`、`rbac`、`audit`、`data-scope`
* 可选聚合：`athena-framework-starter-security-all`（全量场景）

最小化接入（仅认证）：

[source,xml]
----
<dependency>
    <groupId>org.athena</groupId>
    <artifactId>athena-framework-starter-security-auth-core</artifactId>
</dependency>
<dependency>
    <groupId>org.athena</groupId>
    <artifactId>athena-framework-starter-security-token-jwt</artifactId>
</dependency>
----

需要权限鉴权时再追加：

[source,xml]
----
<dependency>
    <groupId>org.athena</groupId>
    <artifactId>athena-framework-starter-security-authorization</artifactId>
</dependency>
----

需要菜单能力时再追加：

[source,xml]
----
<dependency>
    <groupId>org.athena</groupId>
    <artifactId>athena-framework-starter-security-menu</artifactId>
</dependency>
----

== 10. 配置设计

[source,yaml]
----
athena:
  security:
    auth:
      enabled: true
      token-header: Authorization
      token-prefix: Bearer
      ignore-urls:
        - /auth/login
    token:
      type: jwt  # jwt | redis
      jwt:
        enabled: true
        access-token-expire-minutes: 120
      redis:
        enabled: false
        access-token-expire-minutes: 120
    authorization:
      enabled: false
    menu:
      enabled: false
    rbac:
      enabled: false
    audit:
      enabled: false
    data-scope:
      enabled: false
----

说明：

* `auth.enabled` 建议默认开启（认证核心）
* 其他模块默认关闭，按需引入并开启
* 即使配置开启，若未引入模块依赖，也不会生效
